# サービス仕様書

## 概要

白クロスの脳内ソルブ練習と F2L ペア先読み訓練のための、iPhone (Safari) 向け個人用ルービックキューブ練習サービス。

**練習フロー**: シャッフル → 白クロスをソルブ → F2L ペアを見つける → 次のシャッフル (ひたすら白クロスと F2L
ペア探しのみ訓練する)

**対象環境**: iOS Safari のみ。PC・権限拒否等のフォールバック実装は行わない。

## 画面レイアウト (iPhone 縦)

```
┌──────────────────────┐
│ [シャッフル] [やり直す] [スクランブル] [⚙] │  ← ⚙ で設定モーダル
├──────────────────────┤
│                                            │
│                                            │
│      3D Rubik's Cube                       │  ← メイン領域 (スワイプ操作)
│    (やや右面が見える角度)                  │     3面が見える標準持ち角度
│  (ジャイロで微調整して裏面確認)            │
│                                            │
│                                            │
├──────────────────────┤
│ 白クロス解法                               │
│ 解 1: R D F' D2                            │  ← 複数パターン表示
│ 解 2: L' D' F D                            │
│ [◀][▶][▶▶]                             │  ← 一手戻す / 一手進む / 一気に
├──────────────────────┤
│ F2L: [1] [2] [3] [4]                       │  ← ペアトグル (優先順に並ぶ)
└──────────────────────┘
```

**レイアウトの方針**:

- 操作は全てスワイプで行うため、操作ボタンパネルは設けない
- スクランブル手順はモーダル表示 (画面領域を占有しない。見たいときは実キューブを触りたいときなので他要素は隠れてよい)
- キューブは3面が見える角度 (やや右面が見える標準的な持ち方) で表示

## 操作方法

### スワイプ操作 (メイン操作)

キューブの面をスワイプして回転操作を行う。

| 操作     | スワイプ                                 |
|--------|--------------------------------------|
| U / U' | 上面の1行目を左右にスワイプ                       |
| D / D' | 上面の3行目を左右にスワイプ                       |
| L / L' | 上面の1列目を上下にスワイプ                       |
| R / R' | 上面の3列目を上下にスワイプ                       |
| F / F' | 右面の1列目を上下にスワイプ **or** 左面の3列目を上下にスワイプ |
| B / B' | 右面の3列目を上下にスワイプ **or** 左面の1列目を上下にスワイプ |
| 持ち替え   | センターキューブをスワイプ (上下左右)                 |

- M / S ムーブは非対応
- ダブルムーブ (R2 等) はシングルムーブ2回で代替
- スワイプ開始時、対象の面 (レイヤー) を発光させてフィードバック

### ジャイロスコープ (視点微調整)

- `DeviceOrientationEvent.requestPermission()` (iOS Safari)
- デバイスの傾きでキューブの視点を微調整 (見えない側をちょっと覗く用途)
- 操作とは独立した視点制御

## スクランブル

- `cubejs` の `Cube.scramble()` でランダムステートスクランブルを生成
- 「シャッフル」ボタン: 新規スクランブル生成 → キューブに適用
- 「スクランブル」ボタン: スクランブル手順をモーダルで全画面表示
- 「やり直す」ボタン: 同一スクランブルで完成状態からやり直し

## 白クロスソルバー

1. スクランブル適用後、白クロスの最適解を**複数パターン**算出して表示
2. 各解法を回転記号テキストで表示 (例: `R D F' D2`)
3. 再生コントロール:
    - **[◀] 一手戻す**: 1手分キューブを巻き戻す
    - **[▶] 一手進む**: 1手分キューブにアニメーション適用
    - **[▶▶] 一気に再生**: 全手順を連続アニメーション再生
4. 解法パターンの切り替え (解1, 解2, ... をタップで選択)

## F2L ガイド

### ペア定義

白クロス完成後 (白面が下)、4つの F2L スロット:

| # | スロット    | コーナー  | エッジ |
|---|---------|-------|-----|
| 1 | 右前 (RF) | 白-赤-緑 | 赤-緑 |
| 2 | 左前 (LF) | 白-緑-橙 | 緑-橙 |
| 3 | 左後 (LB) | 白-橙-青 | 橙-青 |
| 4 | 右後 (RB) | 白-青-赤 | 青-赤 |

### UI

- ボタン [1] [2] [3] [4] でペアの表示をトグル
- ON にしたペアの構成ピース (コーナー + エッジ) を 3D キューブ上でグロー/発光エフェクトで強調
- **ボタンの並び順 = 優先順** (スコアリングで動的に決定)

### F2L ペア優先順スコアリング

各ペアにスコアを算出し、スコアの高い順に `[1] [2] [3] [4]` を並べる。

#### スコア要素

| 要素                        | 説明                                                        | 重み |
|---------------------------|-----------------------------------------------------------|----|
| **Edge Orientation (EO)** | エッジの向きが正しい (good edge) なら加点                               | 高  |
| **バックスロット優先**             | 対象スロットが背面側 (LB, RB) なら加点。フロントスロットのペアより先にバックスロットを処理する方が効率的 | 中  |
| **ピース視認性**                | ペアを構成するコーナー・エッジが現在の視点から色を判別できるかで加点/減点                     | 中  |

#### ピース視認性の判定ルール

**基本原則**: 3面が見える標準視点 (上面・前面・右面) から色が判別できないピースは優先度を下げる。

**エッジの視認性**:

- 上面・前面・右面にあるエッジ → 視認可能
- 背面・左面・下面のみにあるエッジ → 視認不可
- **消去法ルール**: 同じスロットに属する他の3エッジが全て視認できている場合、残り1エッジは消去法で位置が判明するため「視認可能」として扱う

**コーナーの視認性**:

- 3色のうち2色以上が見えているコーナー → **視認可能として扱う** (見えない3色目を考える力を鍛えるため)
- 1色以下しか見えていないコーナー → 視認不可

#### スコア計算 (暫定値、調整予定)

```
score = eo_bonus + back_slot_bonus + visibility_bonus

eo_bonus       = エッジの向きが正しい ? 30 : 0
back_slot_bonus = バックスロット (LB/RB) ? 20 : 0
visibility_bonus:
  コーナー視認可能 かつ エッジ視認可能 → 40
  コーナー視認可能 かつ エッジ視認不可 → 20
  コーナー視認不可 かつ エッジ視認可能 → 10
  両方視認不可 → 0
```

**同点の場合**: スロット位置順 (RF → LF → LB → RB) で決定。

> 重みの具体的な数値は設定モーダルからいつでも調整できる。上記は初期値。

## 設定 (⚙)

コントロールバーの ⚙ ボタンからモーダルで開く。設定値は localStorage に保存され、リロードしても保持される。

| カテゴリ | 項目               | 初期値   | 説明                             |
|------|------------------|-------|--------------------------------|
| ジャイロ | 感度               | 1.0   | 傾き → 視点回転の倍率 (0.5 で鈍く、2.0 で敏感) |
| ジャイロ | 最大角度             | 30度   | 視点回転の上限角度                      |
| スワイプ | 判定距離             | 20px  | スワイプと判定する最小ドラッグ距離              |
| スワイプ | アニメーション速度        | 300ms | 面回転アニメーションの所要時間                |
| F2L  | EO ボーナス          | 30    | Edge Orientation が正しいときの加点     |
| F2L  | バックスロットボーナス      | 20    | バックスロット (LB/RB) の加点            |
| F2L  | 視認性ボーナス (両方)     | 40    | コーナー・エッジ両方視認可能時の加点             |
| F2L  | 視認性ボーナス (コーナーのみ) | 20    | コーナーのみ視認可能時の加点                 |
| F2L  | 視認性ボーナス (エッジのみ)  | 10    | エッジのみ視認可能時の加点                  |

- 各項目はスライダーで調整
- 変更は即座に反映される
- 「デフォルトに戻す」ボタンで全項目を初期値にリセット

## 検証項目

1. `yarn dev` → ローカル開発サーバーで動作確認
2. `yarn lint` → ESLint エラーゼロ確認
3. `yarn build` → `dist/` 生成確認
4. iPhone 実機確認項目:
    - キューブが世界標準配色で正しく描画される
    - 3面が見える角度で表示される
    - シャッフル → スクランブル適用
    - スクランブルがモーダルで正しく表示される
    - やり直し → 同一スクランブルに戻る
    - スワイプで全ムーブが正しく動く
    - センタースワイプで持ち替えが正しく動く
    - スワイプ開始時の面ハイライトが機能する
    - ジャイロで視点が微調整できる
    - 設定モーダルで各パラメータを変更できる
    - 設定がリロード後も保持される
    - 白クロス解法が複数パターン表示される
    - 一手進む/一手戻す/一気に再生が動く
    - F2L ペアトグルでピースがハイライトされる
    - F2L ペアの並び順がスコアリングで動的に変わる
5. GitHub Pages デプロイ → 本番 URL で動作確認
